# When NOT to Adapt

```
┌─────────────────────────────────────────────────────────────────────┐
│                  WHEN NOT TO ADAPT                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Adaptation is powerful but can go wrong. Hold steady when:         │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ DON'T adapt on a single data point                          │   │
│  │                                                             │   │
│  │ One failed lesson does not equal a pattern. Need 2-3 data  │   │
│  │ points before changing scaffolding or difficulty.           │   │
│  │ Exception: frustration signal always triggers pause.        │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ DON'T change representation every lesson                     │   │
│  │                                                             │   │
│  │ Switching representations is itself disruptive.              │   │
│  │ Only switch when there's clear evidence of struggle.         │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ DON'T let fast learners skip foundations                     │   │
│  │                                                             │   │
│  │ Even fast learners need storage strength. They can          │   │
│  │ go faster, but they can't skip the spaced retrieval.         │   │
│  │ Accelerate by combining, not by omitting.                   │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ DON'T over-scaffold slow learners forever                    │   │
│  │                                                             │   │
│  │ The goal is to GRADUALLY REMOVE scaffolding, not            │   │
│  │ keep it permanently. If they've been at Level 3              │   │
│  │ for 5+ lessons, probe whether they can drop to Level 2.    │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ DON'T confuse topic difficulty with learner aptitude         │   │
│  │                                                             │   │
│  │ A struggle on a genuinely hard concept doesn't mean         │   │
│  │ the learner is "slow." It means the concept is hard.         │   │
│  │ Check: are other learners struggling with this too?          │   │
│  │ (Consult RESOURCES.md for external validation.)             │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ DON'T adapt the mission                                     │   │
│  │                                                             │   │
│  │ Adaptation changes HOW you teach, not WHAT you teach.        │   │
│  │ Mission changes are separate and require explicit             │   │
│  │ user conversation.                                          │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
