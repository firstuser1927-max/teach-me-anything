# Representation Switching Matrix

```
┌─────────────────────────────────────────────────────────────────────┐
│                  REPRESENTATION SWITCHING                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  When a learner struggles with one representation,                  │
│  switch to an alternative. Track which works.                       │
│                                                                     │
│  ┌────────────────┬────────────────────────────────────────────┐   │
│  │ IF STRUGGLING  │ TRY THIS INSTEAD                         │   │
│  │ WITH...        │                                           │   │
│  ├────────────────┼────────────────────────────────────────────┤   │
│  │ Concept-first  │ → Example-first (show it working)         │   │
│  │ (theory before │ → Analogy-first (relate to known thing)   │   │
│  │  practice)     │                                           │   │
│  ├────────────────┼────────────────────────────────────────────┤   │
│  │ Example-first  │ → Concept-first (explain the why)         │   │
│  │ (show then     │ → Visual-first (diagram the pattern)      │   │
│  │  explain)      │                                           │   │
│  ├────────────────┼────────────────────────────────────────────┤   │
│  │ Visual-first   │ → Analogy-first (narrate the diagram)     │   │
│  │ (diagrams,     │ → Code/example-first (make it concrete)   │   │
│  │  mental models) │                                           │   │
│  ├────────────────┼────────────────────────────────────────────┤   │
│  │ Code-first     │ → Concept-first (what's this doing?)      │   │
│  │ (syntax before │ → Example-first (run it, see output)      │   │
│  │  meaning)      │                                           │   │
│  ├────────────────┼────────────────────────────────────────────┤   │
│  │ Analogy-first  │ → Example-first (show the real thing)     │   │
│  │ (metaphors,    │ → Concept-first (drop the metaphor,       │   │
│  │  comparisons)  │   explain directly)                       │   │
│  └────────────────┴────────────────────────────────────────────┘   │
│                                                                     │
│  TRACKING: In LEARNER-PROFILE.md, record:                          │
│  - Which representation was used                                   │
│  - Whether it worked (passed check questions)                      │
│  - Build a preference ranking over time                            │
│                                                                     │
│  NOTE: Some topics have natural best representations.               │
│  A visual-first learner still needs to read code for syntax.        │
│  Adapt the INTRODUCTION, not the practice itself.                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
