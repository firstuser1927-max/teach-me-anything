# Error Pattern Response Strategies

```
┌─────────────────────────────────────────────────────────────────────┐
│              ERROR PATTERN RESPONSE STRATEGIES                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  When you identify an error pattern, apply the matching strategy:   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ CARELESS ERRORS                                             │   │
│  │                                                             │   │
│  │ Pattern: Right answer when prompted to slow down,           │   │
│  │          typos, skipped steps, didn't read fully            │   │
│  │                                                             │   │
│  │ Strategy:                                                   │   │
│  │ • Add "slow down" prompts before check questions            │   │
│  │ • Don't change difficulty or scaffolding                    │   │
│  │ • Add a brief pause/confirmation step in practice           │   │
│  │ • Record as "careless" — not a learning issue               │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ CONCEPTUAL GAP                                              │   │
│  │                                                             │   │
│  │ Pattern: Misses questions on foundational concepts,         │   │
│  │          doesn't understand "why" behind steps              │   │
│  │                                                             │   │
│  │ Strategy:                                                   │   │
│  │ • STOP current progression                                 │   │
│  │ • Identify the missing prerequisite                        │   │
│  │ • Create a focused mini-lesson on the gap                  │   │
│  │ • Promote scaffolding to Level 3                           │   │
│  │ • Don't proceed until gap is closed                        │   │
│  │ • Record gap in learning records                           │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ CONCEPTUAL TRANSFER                                         │   │
│  │                                                             │   │
│  │ Pattern: Understands in the lesson context but can't        │   │
│  │          apply to a new context or slightly different       │   │
│  │          problem                                             │   │
│  │                                                             │   │
│  │ Strategy:                                                   │   │
│  │ • Add transfer practice to lessons (different contexts)    │   │
│  │ • Use analogies that connect to diverse domains            │   │
│  │ • After teaching a concept, immediately ask:               │   │
│  │   "Where else might this apply?"                           │   │
│  │ • Don't change scaffolding — change the practice variety   │   │
│  │ • Record as "transfer" pattern                             │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ OVERGENERALIZATION                                          │   │
│  │                                                             │   │
│  │ Pattern: Applies a rule too broadly, doesn't see           │   │
│  │          boundary conditions or exceptions                  │   │
│  │                                                             │   │
│  │ Strategy:                                                   │   │
│  │ • Add "when does this NOT apply?" to every lesson          │   │
│  │ • Include counter-examples in practice                     │   │
│  │ • Teach boundary conditions explicitly                     │   │
│  │ • Add to check questions: "Which of these is NOT X?"       │   │
│  │ • Record as "overgeneralization" pattern                   │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
