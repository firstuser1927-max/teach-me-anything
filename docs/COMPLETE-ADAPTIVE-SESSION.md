# Complete Adaptive Session Flow

```
┌─────────────────────────────────────────────────────────────────────┐
│                 COMPLETE ADAPTIVE SESSION                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  PHASE 1: OPEN (2-3 minutes)                                       │
│  ─────────────────────────────                                      │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 1. Read LEARNER-PROFILE.md                                  │   │
│  │ 2. Read last 3 learning records                             │   │
│  │ 3. Read NOTES.md for recent observations                     │   │
│  │ 4. Recalibrate profile dimensions with new evidence         │   │
│  │ 5. Calculate difficulty score from last 3 lessons           │   │
│  │ 6. Determine scaffolding level for this session             │   │
│  │ 7. Check: session capacity reached? fatigue? absence?     │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                     │                               │
│                                     ▼                               │
│  PHASE 2: ADAPT & BUILD (varies)                                   │
│  ────────────────────────────────                                    │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 1. Get ZPD skill from standard calculation                  │   │
│  │ 2. Apply difficulty engine decision (elevate/maintain/etc) │   │
│  │ 3. Apply error pattern strategy if needed                   │   │
│  │ 4. Select representation based on profile                   │   │
│  │ 5. Build lesson with selected scaffolding level             │   │
│  │ 6. Add spaced retrieval questions if due                    │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                     │                               │
│                                     ▼                               │
│  PHASE 3: DELIVER & DETECT (varies)                                │
│  ───────────────────────────────────                               │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 1. Deliver lesson to user                                   │   │
│  │ 2. Monitor for real-time signals:                           │   │
│  │    • Confusion → Reframe                                   │   │
│  │    • Boredom → Accelerate                                  │   │
│  │    • Frustration → Ease off                                │   │
│  │    • Flow → Note for elevation                             │   │
│  │    • Fatigue → Wrap up                                     │   │
│  │ 3. Collect check question results                          │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                     │                               │
│                                     ▼                               │
│  PHASE 4: ASSESS & RECORD (2-3 minutes)                            │
│  ────────────────────────────────────────                            │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 1. Classify lesson outcome: pass / struggle / fail          │   │
│  │ 2. Update difficulty score                                  │   │
│  │ 3. Write learning record including:                         │   │
│  │    - Skill gained (if pass)                                 │   │
│  │    - Error pattern observed (if any)                      │   │
│  │    - Signals detected during lesson                         │   │
│  │    - Adaptation triggered                                   │   │
│  │ 4. Create reference doc or glossary entry if ready          │   │
│  │ 5. Check: session capacity remaining?                       │   │
│  │    - Yes → Loop back to PHASE 2                            │   │
│  │    - No → Proceed to close                                 │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                     │                               │
│                                     ▼                               │
│  PHASE 5: CLOSE (2-3 minutes)                                       │
│  ─────────────────────────────                                      │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 1. Update LEARNER-PROFILE.md with session evidence          │   │
│  │ 2. Add row to Adaptive History table                       │   │
│  │ 3. Summarize what was learned                               │   │
│  │ 4. State what's next (adapted for their level)              │   │
│  │ 5. Note any frustration or concerns in NOTES.md             │   │
│  │ 6. Confirm files created/modified                           │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
